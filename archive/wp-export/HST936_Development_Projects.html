Overview

This page contains information for developers in the HST.936 course who are working on group based programming projects. For projects based on the core Sana components, please see the Core section of this page. For projects not based on the Sana core components, please see the Alternative Code Projects section. For additional information on working with Git and the Github site, please see the Reference section at the end of this page.

Core Projects

Workflow

The basic workflow for working with the core component repositories is as follows.
  1. Individual Github acccount for each team member
  2. Single fork of official Sana repository under one team members account
  3. Add all of the team members as collaborators
  4. All team members should make contributions to the fork.
  5. Add any documentation to the Github wiki of your fork
  6. At the end of the course, submit a pull request to the Sana repository from your fork

Detailed Instructions for creating a fork

  1. Login to Github
  2. Open the Sana Github Site
  3. Select the repository page of the component you are working on

    • Android Client 
      https://github.com/SanaMobile/sana.mobile.git
    • MDS 
      https://github.com/SanaMobile/sana.mds.git
    • Encounter Queue Module 
      https://github.com/SanaMobile/sana.openmrs.git
  4. Create a fork of the repository.
  5. Open the Collaborators tab of your new repository's Settings
  6. Add each of your team members as a collaborator
  7. Clone the repository and checkout the hst936 branch to your workstation either through the command line

    git clone https https://github.com/[Github username]/[Repository Path]

    or through Git integration of your IDE. Important Verify the branch with your mentor.
  8. Each team member should edit, test, commit, and push any changes to the hst936 branch of the fork. Mentors will provide additional instructions on frequency of expected commits and any milestones.
  9. At the end of the semester, create a pull request. This can be done ether through the command line or Github web interface.
Please see see the Reference section for more information on working with Git if you are not familiar with it.

Code Conventions

The coding conventions for the Sana repositories in general follow standard best practices for Java and Python. To assist with making your changes adhere to the conventions as the original source, please import the following code formatting styles into your IDE.
  • Mobile Client Code Style for Android Studio
  • MDS Code Style for Aptana Studio
  • Encounter Queue Module Code Style
Instructions
  • Android Studio: 
    File -> Import Settings
    and select the downloaded file.
  • Aptana Studio and Eclipse: 
    Window -> Preferences -> Java -> Code Style -> Code Templates
    and Import the downloaded file.

Alternative Code Projects

For code based projects not based on the core Sana platform components, please perform the following:
  1. Create a Github account if you do not have one
  2. Create a private repository for the code using your Github account.
  3. Add any other students who are working with you on the project as collaborators using their Github account info.
  4. Add Github user hst936 as a collaborator
  5. Fill in the HST.936 Project Github Information form with your Github information and url of the repository.
Please see the Reference section if you have additional questions on working with Git or Github.
